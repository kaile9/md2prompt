# 2youg1的MD2Prompt

[English](README.en.md) · [下载最新版](../../releases) · [设计基准 SPEC](SPEC.md) · [问题反馈（Discussions）](../../discussions) · [提 PR](../../pulls)

> 本项目从设计到实现完全由 [Kimi K3](https://www.moonshot.cn) 搭建，未使用任何其它模型。

**一个 HTML 文件，就是你和 AI 之间的修订工作台。**

双击打开，像 Word 一样改 Markdown；你的每一处改动自动留痕；改完点一下「复制 Prompt」，AI 不用看原文、不用调工具，就能逐条读懂你改了什么、还想让它做什么。

![渲染模式](docs/assets/render.png)

## 为什么做这个

和 AI 远程协作改文案，有三个老毛病：

1. **Word 浪费 token**——AI 原生说 Markdown，没人想贴 `.docx`；
2. **普通 Markdown 渲染器吃掉提示词式 XML 标签**——`<identity>` 这类给 Agent 看的结构标签，渲染器 silently 吞掉，排版全乱；记事本直改图表源码又是转义灾难；
3. **每轮都要重新上传整个文件**——改了三句话，传了三百 KB。

MD2Prompt 的答案是：**协议比编辑器重要**。编辑器只是薄层，真正的产品是一份协议精确的 `Prompt.md` 日记——你的修改、你的批注，实时落盘成 AI 能精确消费的格式。原文是 AI 写的（在它的上下文里），日记只传增量。

## 30 秒上手

1. 去 [Releases](../../releases) 下载 `2youg1-md2prompt.html`（就一个文件，6 MB，依赖全部内嵌，无后端、API 或遥测；编辑本地文件无需联网，远程图片与外链仍按浏览器规则请求网络）；
2. 双击，用 Chrome / Edge 打开，点「打开」选你的 `.md` / `.jsonl` / `.xml`；
3. 直接改。右侧「修订」栏立刻出卡片；
4. 点「复制 Prompt」，粘给 AI。AI 拿到：文档名、BLAKE3 哈希、每条修改（original+alter+行号）、每条批注（命令/建议/讨论）；
5. AI 改完新版本发回来，保存成文件再「打开」——哈希配对成功，**你之前的全部留痕自动恢复**，接着改。

> 误触防护：原文始终干净，每个动作实时落盘。就算直接关网页，打开同目录的 `文档名.prompt.md` 复制内容照样能当提示词用。

## 功能一览

- **编辑即留痕**：替换/插入/删除/调换/批注五类变更自动记录，句级粒度展示（旧句删除线、新句高亮），像 Word 修订但不用点"接受"。
- **批注三型**：选中文字 → 浮卡「✎」（或 `Alt+M`）——**命令**（请 AI 照做）/ **建议**（AI 定夺）/ **讨论**（只聊不改），浮层里一键切换；选段虚线下划线 + 批注钉；侧栏卡片显示引文、可改注、可撤回。
- **块调换**：`Alt+↑/↓` 与相邻块调换（自动记录）；工具轨「⇄」输入行号与任意块调换——swap 自逆，撤回/复活都是再换一次。
- **隐藏 / 撤回（两阶段）**：确认过的点「隐藏」收起；反悔点「撤回」——先预览删除线（可取消），再确认才回滚；撤掉的进墓碑（上限 50），随时「复活」。
- **渲染 / 源码 / 分屏三模式**：渲染所见即所得；源码是 CodeMirror（语法高亮、行号、查找替换）；分屏左源码右预览、**块锚行对齐**（对应块顶对齐，不再是粗糙的比例同步）。批注与行内格式（B/I/S/行内码/链接）**三种模式都能用**。
- **XML 卡片直接改**：`<identity>` 等标签块渲染成卡片，点进卡内直接编辑源文，逐字留痕——不吞标签，也不弹窗整块替换。
- **渲染保真**：Mermaid 图、KaTeX 公式、Markdown 表格、图片（相对路径经目录授权）、脚注弹窗。
- **JSONL 数据集模式**：虚拟化记录卡片流（万行流畅），表单/原始 JSON 双页签编辑——可当 AI 训练数据清洗台。
- **XML 文件源码承载**：`.xml` 整个文件一个代码块，CodeMirror 直改，修订语义干净（整块 replace，大文件自动 patch 省 token）。
- **大文档友好**：超 300KB / 2000 行自动切节；增量重解析让击键入账是 O(变更域) 而非 O(节)（300KB 节实测 341ms → 2.3ms）。
- **三主题 × 两风格**：纯黑 / 云石 / 暖纸 × 极客 / 人文；字号、字重、行距、页宽（任意值可用）、对齐、亮度/对比度、中西文字体栈、首行缩进、行号栏、行引导线——长文案写作可以直接替代 Word。
- **编辑工具**：竖直工具轨（标题/引用/列表/代码块/分割线/链接/图片/调换）+ 选区浮卡 + 可自定义快捷键。
- **导出**：复制 Prompt（省略墓碑）/ 下载 Prompt.md / 下载干净副本 / 导出 PDF（页眉源文件完成时间、页脚导出时间）。

## 说明书

### 打开与保存

- 「打开」走浏览器 File System Access API（Chrome/Edge）；Firefox 降级为上传/下载。文件句柄存在你自己的 IndexedDB，重启后点一下页面即可恢复上次文档。
- 打开时会请求**父目录授权**：用于解析相对路径图片、以及把 `文档名.prompt.md` 日记写到同目录。可以拒绝——拒绝后图片显示占位、日记不自动落盘（仍可手动「下载 Prompt.md」）。
- 每个动作都在 800ms 防抖内自动保存（顶栏「已存/写入中/失败」）。「日记写入失败」与文档保存失败分开提示。

### 修改与留痕

- 改就完了。替换、插入、删除自动成卡；卡片上可「隐藏」（已确认收起，仍导出）、「撤回」（两阶段：先预览，再确认回滚）。
- **调换**：光标放进某段，`Alt+↑/↓` 与相邻段换位；或点工具轨「⇄」输入目标行号，与那一行所在块互换。
- **跳转**：卡片上的「跳转」精确到块；大纲、minimap 点骨架同样可达。

### 批注（note 三型）

- 选中文字 → 浮卡「✎」，或不选直接 `Alt+M`（光标所在块）。浮层顶部选型：
  - **命令 request**：「照这个改」——AI 执行并返回改后文本；
  - **建议 suggest**：「我觉得这样更好，你看着办」——AI 定夺；
  - **讨论 discuss**：「这里我想不通」——AI 只回答，不动文本。
- 同一块再批注 = 编辑原批注（可改文字、改型、改选段）。

### 恢复与配对

- 同目录找到 `文档名.prompt.md` 且哈希一致 → 全部留痕自动恢复。
- 哈希不一致（文档被外部改过）→ 三选：**新基线**（旧记录作废重来）/ **仍尝试恢复** / **忽略**（保留旧日记不动，本次不覆写）。

### 导出给 AI

- **复制 Prompt**：日常主路径——省略墓碑，结构自解释，Opus/K3 级模型直接读懂；
- **下载 Prompt.md**：完整日记（含墓碑）；
- **下载干净副本**：当前文档全文（无留痕）；
- **导出 PDF**：浏览器打印，页眉源文件完成时间、页脚导出时间。

## Prompt.md 长什么样（协议 2.0 一瞥）

```xml
---
protocol: md2prompt/2.0.0
doc: 宪法中译.md
doc-hash: blake3:9f2c…
base-hash: blake3:71be…
changes: 3
---

# 修改记录 · 宪法中译.md
<!-- 这是人对文档的修改日记（原文在你上下文中；行号基于当前文档）：revise/swap=人已改完（理解即可）；note=人请你处理（request=照做，suggest=定夺，discuss=讨论）。 -->

<changes>
<note n="1" line="56" request="这句太长，请拆成两句并补一个例子。"><range>本宪法旨在指导 Claude 的价值观与行为……</range></note>
<revise n="2" line="102"><original>…原文…</original><alter>…新文…</alter></revise>
<swap n="3" a="40" b="52"><first>甲块首行</first><first>乙块首行</first></swap>
</changes>
```

- 单流 `<changes>` 按 `n`（修改顺序）排列，`n` 跨导出稳定（Agent 端缓存友好）；
- note 三型收进属性；revise 缺 `original` = 新增、缺 `alter` = 删除；大块修改走 `<del>/<ins>` patch 形（省 token）；
- 墓碑只留在日记文件里，复制时自动省略。

完整协议见 [SPEC.md](SPEC.md) §3（唯一权威设计基准，含全部修订记录）。

## 更新日志

### 2.0.0（vs 1.5.2）

**协议破坏性变更（不兼容 1.x 日记）**

- 协议 2.0：单流 `<changes>` 按 `n` 排序；`<request>/<edit>` 退役，改为 `note`（request/suggest/discuss 三型）与 `revise`（original+alter）；`time` 属性删除；复制版省略墓碑区段与计数行；结构自解释（无需教学前言，Opus/K3 级模型直接读懂）。
- move 退役，swap 接任：新增「X 行和 X 行调换」命令（Alt+↑/↓ 相邻、工具轨 ⇄ 任意行）。

**缺陷修复**

- 长文档修订错行/框选错锚（标签区域并块根治）；一句话被 `SKILL.md` 等腰斩（句首启发式）；页宽 >60 显示失败（#doc 显式宽度）；分屏行对齐（块锚同步）；源码/分屏可批注可改；XML 修订语义（源码承载单块 diff）；「忽略」后日记被首次编辑覆写；math_block 的 Ctrl+A 逃逸；日记失败不可见。

**性能**

- 击键入账增量重解析 ≈148×（300KB 节 341ms→2.3ms）；diff 护栏总量上限；万行大纲不再每击键重建。

**工程**

- 单测 154 → 211 例（13 文件）；E2E 门禁 6 → 10 套（新增页宽/分屏对齐/源码批注/XML 四套）；27 个源文件 MPL-2.0 SPDX 头；index.html 加 CSP；safeUrl 表驱动 XSS 回归入库。

### 1.5.2

- 初始开源版本（v1.5.1）+ PR #2 文件写入完整性修复（跨目标/跨通道写入串行化）。

## FAQ

**为什么不做成 VS Code 插件？** 因为要服务的不是程序员场景，而是文案场景：双击即用、不改工作流、不装运行时。一个 HTML 文件是门槛最低的形态。

**AI 不会用工具也能配合吗？** 能。这正是设计目标：Prompt.md 里的一切都是自解释文本，AI 读完就知道该改哪、怎么改；它返回新全文即可，不需要任何函数调用。

**数据去哪了？** 应用不向服务端上传文档，没有服务端 API 或遥测（CSP 层面 `connect-src 'none'` 互相印证）；文件读写走浏览器 File System Access API，句柄存在你自己的 IndexedDB。文档中的远程图片会由浏览器按其 URL 加载，点击外链也会访问对应网站。

**2.0 的日记和 1.x 兼容吗？** 不兼容（破坏性升级，本地小文件无历史包袱）。旧 1.x 日记请在 1.5.2 里处理完再升级。
